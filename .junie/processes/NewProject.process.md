## Steps

### Define Project

1. Ask for a brief description of the project.
2. Create a `documentation/project_specification.md` file and start populating it with the information.
2. Remove everything that isn't essential to the project to make a mvp (minimum viable product).
3. Defining the tech stack and add that to the project_specification.md file.

### Define Architecture

1. Reread the project specification and define the architecture.
2. Create a `documentation/architecture_specification.md` file and start populating it with the information.
3. Define the architecture and add that to the architecture_specification.md file.

### Create a to-do list

1. read both files (project_specification.md and architecture_specification.md) and come up with 10-20 tasks to complete the project. order the tasks chronologically.
